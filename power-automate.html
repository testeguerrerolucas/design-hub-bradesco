Sobre o Power Automate

O Power Automate (anteriormente Microsoft Flow) é uma plataforma da Microsoft que permite criar fluxos de trabalho automatizados entre seus aplicativos e serviços favoritos para sincronizar arquivos, receber notificações, coletar dados e muito mais.

Com uma interface visual intuitiva de arrastar e soltar, o Power Automate permite que qualquer pessoa, independentemente de suas habilidades técnicas, crie automações poderosas que economizam tempo e reduzem tarefas manuais e repetitivas.

No Bradesco, o Power Automate é utilizado para uma variedade de casos de uso, desde a automação de processos de aprovação até a integração de diferentes sistemas e plataformas.

Principais Recursos

  • 1
    Interface visual intuitiva

    Criação de automações com interface drag-and-drop, sem necessidade de conhecimentos avançados de programação.

  • 2
    Centenas de conectores

    Integração com mais de 300 serviços e aplicativos, incluindo todos os produtos Microsoft e muitos serviços de terceiros.

  • 3
    Automação robótica (RPA)

    Automação de tarefas em aplicativos da web e desktop que não possuem APIs disponíveis.

  • 4
    Templates prontos

    Biblioteca de modelos pré-configurados para casos de uso comuns, acelerando a implementação.

  • 5
    Gatilhos automáticos

    Inicie fluxos com base em eventos como novos e-mails, atualizações de banco de dados ou atividades em redes sociais.

  • 6
    Controle de fluxo avançado

    Implemente condições, loops e tratamento de erros para criar fluxos complexos e robustos.

Quando usar o Power Automate?

Ideal para

  • Integração com produtos Microsoft 365
  • Fluxos de aprovação
  • Notificações e alertas
  • Automação de tarefas simples a médias
  • Equipes com pouca experiência técnica
  • Automação de processos de negócios
  • Integração entre sistemas modernos
  • Processamento e transformação de dados

Não recomendado para

  • Processos muito complexos
  • Automações de alto volume
  • Sistemas legados sem API nem UI
  • Processamento de dados muito intensivo
  • Soluções que exigem latência mínima
  • Casos que exigem manipulação visual avançada
  • Interações físicas com hardware

Nota de Licenciamento

O Power Automate está incluído no Microsoft 365, mas fluxos mais complexos e conectores premium podem exigir licenças adicionais. Consulte a equipe de Low Code Hub para avaliar suas necessidades.

Recursos do Power Automate

Cloud Flows

Automatize processos entre mais de 300 serviços e aplicativos, desde Microsoft 365 até aplicativos de terceiros, utilizando uma interface de design visual.

Desktop Flows (RPA)

Automatize aplicações e processos baseados em desktop utilizando gravação de UI, automatizando tarefas realizadas em interfaces de usuário.

Process Advisor

Identifique processos que podem ser automatizados e receba recomendações sobre onde aplicar a automação para maximizar o impacto.

Business Process Flows

Crie processos guiados para padronizar operações de negócios, garantindo que todos sigam as mesmas etapas em processos importantes.

APIs e Conectores

O Power Automate oferece centenas de conectores para integração com diferentes sistemas. Abaixo estão alguns exemplos de como interagir com eles.

HTTP Chamada REST básica

POST https://api.powerautomate.com/v2.0/flows

Cria um novo fluxo no Power Automate usando a API HTTP.

// Exemplo de requisição para criar um fluxo
const requestOptions = {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + token 
  },
  body: JSON.stringify({
    name: "Meu Novo Fluxo",
    definition: {
      triggers: {
        manual: {
          type: "Request",
          kind: "Http",
          inputs: { schema: {} }
        }
      },
      actions: {
        Send_an_email: {
          type: "ApiConnection",
          inputs: {
            host: { connection: { name: "@parameters('$connections')['shared_office365']['connectionId']" } },
            method: "post",
            body: {
              To: "exemplo@bradesco.com.br",
              Subject: "Notificação Automática",
              Body: "Este é um email automático do Power Automate"
            }
          }
        }
      }
    }
  })
};

fetch("https://api.powerautomate.com/v2.0/flows", requestOptions)
  .then(response => response.json())
  .then(data => console.log(data));

Connector SharePoint

GET /datasets/{site}/tables/{lista}/items

Obtém itens de uma lista do SharePoint.

// Fluxo para obter itens de uma lista do SharePoint
{
  "type": "flow",
  "trigger": {
    "type": "Request",
    "kind": "Http"
  },
  "actions": {
    "Get_items": {
      "type": "ApiConnection",
      "inputs": {
        "host": {
          "connection": {
            "name": "@parameters('$connections')['sharepoint']['connectionId']"
          }
        },
        "method": "get",
        "path": "/datasets/@{encodeURIComponent('https://bradesco.sharepoint.com/sites/LowCodeHub')}/tables/@{encodeURIComponent('ListaNome')}/items"
      }
    }
  }
}

Connector Microsoft Teams

POST /v3/teams/{team}/channels/{channel}/messages

Envia mensagens para um canal no Microsoft Teams.

// Fluxo para enviar mensagem ao Teams
{
  "type": "flow",
  "trigger": {
    "type": "Schedule",
    "inputs": {
      "schedule": {
        "frequency": "Day",
        "interval": 1
      }
    }
  },
  "actions": {
    "Post_message": {
      "type": "ApiConnection",
      "inputs": {
        "host": {
          "connection": {
            "name": "@parameters('$connections')['teams']['connectionId']"
          }
        },
        "method": "post",
        "body": {
          "messageBody": "Lembrete: Reunião diária às 10:00",
          "recipient": {
            "groupId": "teamId",
            "channelId": "channelId"
          }
        },
        "path": "/v3/teams/@{encodeURIComponent('teamId')}/channels/@{encodeURIComponent('channelId')}/messages"
      }
    }
  }
}

HTTP Acesso a API Externa

GET https://api.externa.com/recursos

Conecta a uma API externa usando a ação HTTP no Power Automate.

// Fluxo para acessar API externa
{
  "type": "flow",
  "trigger": {
    "type": "Recurrence",
    "inputs": {
      "frequency": "Hour",
      "interval": 1
    }
  },
  "actions": {
    "HTTP": {
      "type": "Http",
      "inputs": {
        "method": "GET",
        "uri": "https://api.externa.com/recursos",
        "headers": {
          "Content-Type": "application/json",
          "Authorization": "Bearer @{variables('ApiKey')}"
        }
      }
    },
    "Parse_JSON": {
      "type": "ParseJson",
      "inputs": {
        "content": "@body('HTTP')",
        "schema": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "id": { "type": "string" },
              "nome": { "type": "string" },
              "valor": { "type": "number" }
            }
          }
        }
      }
    }
  }
}

Cria um fluxo de aprovação para tomada de decisões.

// Fluxo para criar aprovação
{
  "type": "flow",
  "trigger": {
    "type": "Request",
    "kind": "Http"
  },
  "actions": {
    "Create_approval": {
      "type": "ApiConnectionWebhook",
      "inputs": {
        "host": {
          "connection": {
            "name": "@parameters('$connections')['approvals']['connectionId']"
          }
        },
        "body": {
          "approval": {
            "requestor": "usuario@bradesco.com.br",
            "itemName": "Solicitação de Acesso",

  }

Melhores Práticas

Recomendações para desenvolver fluxos eficientes e sustentáveis no Power Automate.

Nomenclatura padronizada

Utilize um padrão de nomenclatura claro e consistente para os fluxos, facilitando a manutenção e a identificação do propósito de cada automação.

Exemplo: [Departamento]-[Processo]-[Ação]
TI-OnboardingDev-AprovacaoAcesso

Documentação incorporada

Documente o fluxo utilizando descrições em cada ação e incluindo comentários explicativos. Isso facilita a manutenção por outros membros da equipe.

Dica: Adicione descrições em todas as ações importantes e use a ação "Adicionar comentário" para documentar seções complexas do fluxo.

Tratamento de erros

Implemente tratamento de erros adequado para lidar com falhas na execução. Utilize ações de "Scope" com configurações de tratamento de erro para isolar e gerenciar falhas.

Recomendação: Configure notificações de erro e registre detalhes da falha em um sistema de log para facilitar a resolução de problemas.

Segmentação de fluxos complexos

Divida automações complexas em fluxos menores e mais gerenciáveis que se comunicam entre si. Isso melhora a manutenção e o desempenho.

Técnica: Use gatilhos HTTP para permitir que um fluxo chame outro, passando parâmetros entre eles.

Variáveis para valores reutilizáveis

Utilize variáveis para armazenar valores que são utilizados múltiplas vezes no fluxo, facilitando a manutenção e reduzindo a chance de erros.

Minimize o uso de expressões complexas

Evite expressões muito longas e complexas diretamente nas ações. Se necessário, divida o processamento em várias etapas usando variáveis intermediárias.

Controle de concorrência em loops

Em ações "Apply to each", configure o controle de concorrência para otimizar o desempenho sem sobrecarregar os sistemas de destino.

Configuração: Para integrações com APIs externas, limite a concorrência para evitar erros de limitação de taxa (rate limiting).

Minimize chamadas de API desnecessárias

Armazene resultados de chamadas de API em variáveis quando precisar reutilizá-los, em vez de fazer várias chamadas para o mesmo recurso.

Use ambientes de desenvolvimento e produção

Desenvolva e teste os fluxos em ambientes separados antes de implantá-los em produção. Isso minimiza riscos e interrupções.

Implemente monitoramento proativo

Configure mecanismos de alerta e monitoramento para ser notificado sobre falhas ou problemas de desempenho nos fluxos críticos.
